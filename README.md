# 🦆 덕고치 (Duckgotchi)

**개발하다 막힐 때 옆에 두고 키우는 데스크탑 오리.** 다마고치처럼 놀아주고 키우는 오리인데, 막히면 말도 통해요.

화면 위에 늘 떠 있어요. 심심하면 드래그해서 던지거나 더블클릭해서 반응을 구경하고,
코드가 안 풀릴 땐 오리한테 그냥 말을 걸면 됩니다 — 진짜 AI가 대신 답해줘요.
같이 놀아줄수록 경험치가 쌓여서, 머리부터 발끝까지 원하는 대로 키우고 꾸밀 수 있어요.

> **다마고치 × 고무오리.** 예전 다마고치처럼 옆에 두고 키우는 재미에,
> 막힌 문제를 오리한테 설명하다 스스로 답을 찾는 **고무오리 디버깅**을 더했어요 —
> 덕고치는 그 고무오리가 진짜로 대답까지 해주는 버전이에요.

## ✨ 뭘 할 수 있나요

- **항상 화면 위** — 여러 모니터를 자유롭게 넘나들며 바탕화면 맨 위에 떠 있어요. 오리 밖 빈 공간을 클릭하면 그대로 아래 창으로 통과돼서 작업에 방해되지 않아요.
- **말 걸면 대답해요** — 막힌 문제를 적으면, 내가 골라 둔 AI가 답해줘요.
- **가지고 놀기** — 던지고, 튕기고, 더블클릭하면 14가지 반응 중 하나를 랜덤으로 보여줘요.
- **키우는 재미** — 놀수록 경험치(XP)가 쌓이고, 그걸로 오리의 곳곳을 하나씩 꾸며요.

## 🚀 시작하기

**필요한 것:** Windows PC · [Node.js](https://nodejs.org) (미리 설치돼 있어야 해요)

1. 내려받은 `duck-widget` 폴더를 열어요.
2. **`run.bat` 을 더블클릭**하면 끝! 처음 한 번은 준비 작업 때문에 조금 걸리고, 다음부턴 바로 떠요.

터미널이 편하다면:

```
npm install   # 최초 1회만
npm start
```

**끄기:** 오리 우상단 **"종료"** 버튼을 누르거나, 어디서든 **`Ctrl+Shift+Q`**.

## 🖱️ 조작법

| 이렇게 하면 | 이렇게 돼요 |
|---|---|
| 오리를 **짧게 클릭** | 말풍선이 떠서 대화를 시작해요 |
| **더블클릭** | 폴짝·스핀·젤리·하트비트 등 14가지 반응 중 랜덤 (+ 효과음) |
| **드래그** | 오리를 집어 던져요 — 미끄러지다 화면 끝에서 통통 튕겨요 |
| **꾹 눌러 0.4초 기다렸다가** 드래그 | 제자리에서 빙글 돌려요 |
| **우클릭** | 설정 창을 열어요 (AI 고르기 · 성격 추가 · 꾸미기) |
| 좌상단 **✨ 경험치 배지** 클릭 | 꾸미기 화면으로 바로 이동 |

## 💬 오리랑 대화하기

오리를 클릭하고, 뜬 말풍선에 막힌 걸 적어 Enter를 누르면 돼요. 대화는 문맥이 이어져서 계속 이어 물어볼 수 있어요.
답변에 참고할 링크가 있으면, 오리 머리 위에 구름 모양 말풍선으로 따로 띄워줘요.

말풍선엔 **최신 답변 한 줄**만 떠요(원래의 오리 말풍선 느낌). 주고받은 **전체 기록**은 우클릭 → 설정 → **대화내역** 탭에서 스크롤로 보고, 거기서 **초기화**할 수 있어요.
잠깐 안 건드리면 말풍선은 **저절로 닫혀요** — 닫히는 시간은 설정에서 조절해요(0으로 두면 안 닫힘).

**어떤 AI가 답하게 할지는 우클릭 → 설정에서 골라요.** 넷 중 편한 걸 쓰면 됩니다:

- 🟢 **가장 쉬운 길 — 이미 쓰는 AI 그대로** — 컴퓨터에 **Claude Code**나 **Codex**가 깔려 있고 로그인돼 있으면, 그걸 그대로 써요. 키 입력도, 추가 요금도 없어요.
- 🔑 **API 키로 연결** — OpenAI · Anthropic · Google Gemini · OpenRouter · Groq · xAI. 설정에 키와 모델만 넣으면 끝.
- 💻 **내 컴퓨터 안에서** — Ollama · LM Studio 같은 로컬 AI. 키가 필요 없어요.
- ⚙️ **그 외** — 위 목록에 없는 서비스도 주소·모델·(필요하면)키를 직접 넣어 연결할 수 있어요.

> 🔒 **키는 안전해요.** 입력한 API 키는 이 PC에만 암호화돼 저장되고, 바깥으로 전송되거나 어딘가에 올라가지 않아요.

오리 기본 성격은 **"건방지고 절대 안 지는 러버덕"** — 이건 항상 켜져 있어요.
여기에 더 시키고 싶은 말투나 규칙이 있으면 설정에서 덧붙일 수 있어요(안 적으면 기본 성격 그대로).

## 🎨 오리 키우고 꾸미기

- 질문 한 번, 더블클릭 반응 한 번마다 **경험치 +1**.
- 우클릭 → **꾸미기** 탭에서 원하는 부위에 경험치를 투자해 레벨을 올려요. 머리·몸통·부리·눈·날개·다리·발·꼬리·볼터치·머리장식·의상·등장식·재질까지 **17군데**를 손볼 수 있어요.
- 레벨이 오를 때마다 그 부위의 새 모양과 색이 하나씩 열려요. 한 번 연 건 계속 남아서 자유롭게 조합할 수 있어요 — 메이플 장비창처럼 겹쳐 입히는 느낌이에요.
- 17군데를 전부 만렙까지 채우면 **Duck Code** 입력창이 열려서, 오리 전체 스타일을 직접 정밀하게 조합할 수 있어요.
- 설정 안 **쿠폰 입력** 칸에 코드가 있으면 넣어서 꾸미기 특전을 열 수 있어요.

## 🐤 잘 안 되나요?

- **클릭해도 반응이 없어요** → 오리 몸통 위를 정확히 눌러 보세요. 몸 밖 빈 공간은 일부러 클릭이 통과되도록 해뒀어요.
- **오리가 안 보여요** → 다른 모니터에 가 있을 수 있어요. 처음엔 화면 가운데쯤에 나타나요.
- **대답이 안 와요** → 설정에서 고른 AI가 준비돼 있는지 확인해요. (CLI는 로그인, API는 키 입력, 로컬 AI는 서버 실행이 필요해요.)

## 🛠️ 개발자 메모

프레임워크도 빌드 단계도 없어요. `index.html` 안의 단일 `<script type="module">` 이 Chromium 에서 그대로 실행돼요.

**구조**

- `main.js` — Electron 메인. 창(투명·클릭스루·전 모니터) 생성, `claude`/`codex` CLI 및 HTTP API 호출, IPC.
- `preload.js` — `contextBridge` 로 `window.duckAPI` 노출.
- `index.html` — 렌더러 전체(마크업·CSS·앱 로직). SVG 오리·물리·말풍선·설정/대화내역/꾸미기 패널·쿠폰까지.

**빌드가 없어서 문법 오류는 실행할 때만 드러나요.** `index.html` 을 손댔으면 스크립트 블록을 Node 로 먼저 검증:

```bash
node -e "const fs=require('fs');const m=fs.readFileSync('index.html','utf8').match(/<script type=\"module\">([\s\S]*?)<\/script>/);fs.writeFileSync('check.mjs',m[1].replace(/^import .*;$/gm,''))"
node --check check.mjs && rm check.mjs
```

(`main.js` 는 `node --check main.js` 로 충분.)

**대화 · 쿠폰**

- 대화기록은 `localStorage["duckConvo"]` 에 쌓이고, 말풍선은 최신 한 줄만 표시 — 전체는 설정 "대화내역" 탭.
- 세션 문맥: CLI 는 `sessionId` 로 resume, HTTP API 는 `apiHistory` 재생.
- 쿠폰은 소스에 **해시만**(`COUPONS` 테이블). 발급·운영은 `for_운영자.md`(로컬 전용) 참고.

**문서 지도**

- `CHANGELOG.md` — 요청별 변경 이력.
- `CLAUDE.md` — AI 어시스턴트용 심화 가이드(로컬 전용, 미배포).
- `for_운영자.md` — 쿠폰 발급·운영(로컬 전용, 미배포).
